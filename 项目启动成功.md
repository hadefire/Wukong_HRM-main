# Wukong_HRM 项目启动成功

## 项目概览

**Ever人力资源管理系统** (Wukong HRM) 是一个前后端分离的人力资源管理平台。

### 技术栈

**后端：**
- Spring Boot 2.7.15
- Spring Cloud Alibaba 2021.0.5.0
- MyBatis Plus 3.5.3.2
- MySQL 8.0
- Redis 7.2
- Java 17

**前端：**
- Vue 2.5.17
- Element UI 2.15.10
- Node.js 16.20.2

## 核心功能模块

1. ✅ **员工组织管理** - 最优配置人才资源
2. ✅ **考勤管理** - 增加员工行动力
3. ✅ **薪资管理** - 助力企业人才战略
4. ✅ **社保管理** - 给与员工充足的保障
5. ✅ **绩效管理** - 调动全员积极性
6. ✅ **员工自助** - 员工所有相关信息一览无余

## 常用操作命令

### 查看后端日志
```bash
tail -f /root/Wukong_HRM-main/backend.log
```

### 停止后端服务
```bash
kill 1028372
```

### 重启后端服务
```bash
cd /root/Wukong_HRM-main
java -jar hrm/hrm-web/target/hrm-web-0.0.1-SNAPSHOT.jar > backend.log 2>&1 &
```

### 停止前端服务
在前端启动的终端按 `Ctrl+C`

### 重启前端服务
```bash
cd /root/Wukong_HRM-main/ux
npm run dev
```

### 重新构建后端
```bash
cd /root/Wukong_HRM-main
mvn clean package -DskipTests -pl hrm/hrm-web -am
```

### 构建前端生产版本
```bash
cd /root/Wukong_HRM-main/ux
npm run build
```

## 故障排查

### 后端无法启动
1. 检查日志: `tail -f backend.log`
2. 检查数据库连接: `mysql -h 127.0.0.1 -u root -p'q*6!TJ7nqRB4'`
3. 检查Redis连接: `redis-cli -h 127.0.0.1 -p 6379 -a '3e%BPY!fMdCZ' ping`

### 前端无法访问
1. 检查前端进程: `netstat -tlnp | grep 8090`
2. 查看编译日志: 检查终端输出
3. 清除缓存重新安装: `cd ux && rm -rf node_modules && npm install`

### API调用失败
1. 检查后端是否正常运行: `netstat -tlnp | grep 28080`
2. 检查浏览器控制台网络请求
3. 确认代理配置: `ux/vue.config.js` 中的 proxy 配置
