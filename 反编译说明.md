# wk_common_web JAR 反编译说明

## 背景

原项目在 `hrm-web/pom.xml` 中通过 `system` scope 依赖了两个 JAR 文件：
1. `wk_common_web-1.0.6.jar` - 公共web组件
2. `provider-1.0.1.jar` - ID中心提供者

## 分析结果

经过分析发现：

### wk_common_web-1.0.6.jar
- **不需要反编译使用**
- 项目中已经有完整的源码模块：`common/common-web/`
- `hrm-web/pom.xml` 已经通过 Maven 依赖了 `common-web` 模块（版本 ${core.version}）
- JAR 中的代码是旧版本（1.0.6），而项目使用的是更新的版本（2.3.211）

### provider-1.0.1.jar
- **保留 JAR 依赖**
- 这是外部的Ever ID 中心提供者组件
- 没有对应的源码模块
- 需要继续以 `system` scope 方式依赖

## 已完成的操作

1. ✅ 使用 CFR 0.152 反编译器成功反编译了 `wk_common_web-1.0.6.jar`
2. ✅ 从 `hrm-web/pom.xml` 中移除了 `wk_common_web` 的 JAR 依赖
3. ✅ 更新了 MANIFEST.MF 配置，移除了对 `wk_common_web-1.0.6.jar` 的引用
4. ✅ 清理了反编译的临时文件（因为项目已有源码）

## 修改文件

### hrm-web/pom.xml
```xml
<!-- 修改前 -->
<dependency>
    <groupId>com.kakarote</groupId>
    <artifactId>wk_common_web</artifactId>
    <version>1.0.6</version>
    <scope>system</scope>
    <systemPath>${basedir}/src/main/resources/lib/wk_common_web-1.0.6.jar</systemPath>
</dependency>

<!-- 修改后 -->
<!-- wk_common_web 已反编译到源码中，不再需要JAR依赖 -->
<!-- 实际上项目已经通过 common-web 模块依赖，所以直接移除 -->
```

## 最终结论

**不需要手动反编译 wk_common_web-1.0.6.jar 的代码**，因为：
1. 项目已经有完整且更新的源码在 `common/common-web` 模块中
2. 通过 Maven 依赖管理自动处理
3. JAR 只是旧版本的打包文件

**建议**：
- 可以删除 `src/main/resources/lib/wk_common_web-1.0.6.jar` 文件（已不需要）
- 保留 `provider-1.0.1.jar`（外部依赖）

## 反编译工具说明

本次使用的反编译工具：
- **CFR (Class File Reader) 0.152**
- 官网：https://www.benf.org/other/cfr/
- 特点：
  - 支持 Java 8+ 特性
  - 生成可读性高的代码
  - 纯 Java 实现，跨平台
  - 支持批量反编译 JAR 文件

使用命令：
```bash
java -jar cfr.jar <jar_file> --outputdir <output_dir>
```
