# 前端功能分析

本文从用户前端视角分析 `ux` 前端项目的功能模块和功能点。分析依据主要来自：

- 前端入口与工程配置：`ux/package.json`、`ux/src/main.js`、`ux/vue.config.js`
- 路由与权限：`ux/src/router/index.js`、`ux/src/router/router.js`、`ux/src/router/modules/hrm.js`、`ux/src/router/modules/admin.js`、`ux/src/router/modules/user.sync.js`、`ux/src/permission.js`、`ux/src/store/modules/permission.js`
- 页面目录：`ux/src/views`
- API 模块：`ux/src/api`

## 1. 项目定位与前端技术栈

### 1.1 项目定位

前端项目位于 `ux` 目录，是一个基于 Vue 2 的 HRM 人力资源管理系统前端。当前可见主业务集中在：

- HRM 人力资源业务端
- HRM 员工自助端
- 系统管理后台
- 个人中心
- 登录、权限、文件导入导出、系统消息、帮助中心等通用能力

代码中仍保留了较多 5kCRM / OA / CRM 相关 API、组件和消息入口，但从当前路由配置看，实际注册到前端菜单的主模块只有 `hrm`、`manage` 和同步路由中的 `person`。

### 1.2 技术栈

- Vue 2.5.17
- Vue Router 3.0.1
- Vuex 3.0.1
- Element UI 2.15.10
- Axios 1.3.1
- ECharts 5.4.0
- FullCalendar
- vue2-org-tree
- el-bigdata-table
- xlsx / file-saver / print-js 等导入导出与打印相关能力

### 1.3 运行与接口代理

`ux/package.json` 中定义：

- `npm run dev`：启动开发服务
- `npm run build`：生产构建
- `npm run build:test`：测试模式构建
- `npm run preview`：本地预览构建产物
- `npm run lint`：修复 JS/Vue lint 问题

开发服务端口来自 `ux/vue.config.js`：

- 默认前端开发端口：`8090`
- 开发代理目标：`http://localhost:28080`
- `VUE_APP_BASE_API` 在 `.env.development`、`.env.production`、`.env.test` 中均为空字符串，表示请求默认走同源路径，由开发代理或后端静态资源承载处理。

## 2. 前端入口与用户访问路径

### 2.1 应用入口

入口文件为 `ux/src/main.js`，主要完成：

- 注册 Element UI、全局样式、全局组件、全局过滤器、全局指令
- 注册虚拟滚动、组织树、文件上传、文件选择、文件预览、导入、导出等通用插件
- 注入全局配置 `WKConfig`
- 注入权限判断方法 `$auth`
- 注入金额单位工具 `$unit`、`$money`
- 注入用户、头像、部门展示工具 `$getUserName`、`$getUserImg`、`$getDeptName`
- 加载缓存
- 设置本地登录模式 `window.__localAuth = true`
- 引入 `permission.js` 做路由守卫和动态路由加载

### 2.2 登录入口

登录页面：`ux/src/views/login/index.vue`

主要功能：

- 展示企业名称、企业登录 Logo
- 从系统配置接口读取公司名称与登录 Logo
- 支持账号密码登录组件 `LoginByPwd`
- 登录 API：`/auth/login`
- 登录成功后保存 Token，并加载用户信息、权限和动态路由

相关文件：

- `ux/src/api/login.js`
- `ux/src/views/login/component/LoginByPwd.vue`
- `ux/src/store/modules/user.js`

### 2.3 全局路由入口

基础静态路由位于 `ux/src/router/index.js`：

- `/login`：登录页
- `/404`：404 页面
- `/noAuth`：无权限页面
- `/link`：外链/审批相关隐藏入口
- `/perview`：预览隐藏入口
- `/person/index`：个人中心，同步路由引入

动态业务路由由 `ux/src/router/router.js` 自动扫描 `ux/src/router/modules/*.js` 生成。

当前动态路由模块：

- `hrm.js`：人力资源模块
- `admin.js`：系统管理模块，内部类型映射为 `manage`
- `user.sync.js`：个人中心同步路由

### 2.4 权限与菜单生成

权限控制链路：

1. 用户访问页面。
2. `ux/src/permission.js` 检查本地 Token。
3. 无 Token 跳转 `/login`。
4. 有 Token 时请求用户信息与权限。
5. `store.dispatch('getAuth')` 调用 `adminIndexAuthListAPI()` 获取权限树。
6. `store.dispatch('GenerateRoutes', auths)` 根据权限过滤路由。
7. `router.addRoutes()` 动态注册可访问页面。
8. 顶部菜单和侧边菜单根据 `hrmRouters`、`manageRouters` 生成。

权限判断规则：

- 路由 `meta.requiresAuth` 表示需要权限。
- `meta.permissions` 表示按权限路径逐级判断，例如 `['hrm', 'salary', 'index']`。
- `meta.permissionList` 表示多个权限满足其一即可。
- `meta.authFun` 用于自定义权限判断。
- 当前本地模式下 `window.__localAuth` 为 true 时，`checkAuth()` 直接返回 true，因此开发/本地环境会弱化权限过滤。

## 3. 整体前端模块地图

从用户可进入的页面看，当前前端主要有以下模块：

| 一级模块 | 入口/路径 | 用户视角说明 |
| --- | --- | --- |
| 登录 | `/login` | 账号密码登录、企业 Logo 和名称展示 |
| HRM 管理端 | `/hrm/...` | 人力资源管理人员或管理者使用的人事、招聘、考勤、社保、薪资、绩效功能 |
| HRM 员工端 | `/hrm/self-server/...` | 普通员工查看个人档案、社保、工资条、考勤、绩效 |
| 系统管理 | `/manage/system/...` | 管理员维护用户、角色、菜单、审批、HRM 参数与规则 |
| 个人中心 | `/person/index` | 当前用户个人信息、账号密码 |
| 隐藏/辅助入口 | `/link`、`/perview`、`/404`、`/noAuth` | 审批链接、预览、异常页 |

## 4. HRM 管理端功能模块

HRM 管理端路由主要来自 `ux/src/router/modules/hrm.js`，布局为 `ux/src/views/layout/HrmLayout.vue`。

`HrmLayout` 的用户体验特点：

- 顶部显示 HRM 导航。
- 部分模块存在左侧二级菜单。
- 管理端和员工端可切换，按钮文案为“进入员工端 / 进入管理端”。
- 当用户具备管理或上级角色时展示端切换按钮。
- 支持“快速创建”员工入口，依赖员工保存权限。

## 13. 前端功能完整性判断

从路由和页面实现看，当前项目已经具备一套完整 HRM 系统前端：

- 有登录和权限加载。
- 有 HRM 管理端。
- 有 HRM 员工端。
- 有系统设置后台。
- 有个人中心。
- 有导入、导出、文件、审批、字段配置等通用能力。

但也存在以下特征：

- 当前前端是从 5kCRM 系统裁剪/改造而来，代码中保留了 CRM/OA/BI/Call 等大量遗留状态、API 和组件。
- 当前菜单只从 `router/modules` 中的 `hrm.js`、`admin.js` 和 `user.sync.js` 生成，未注册 CRM/OA 主路由。
- `main.js` 中开启了本地登录模式，开发/本地环境下权限过滤会被弱化。
- `Navbar.vue` 中系统消息、公告、帮助按钮等入口有代码但被注释，属于保留功能或暂未开放功能。
- `admin.js` 中 `/manage/hrm` 老后台路由保留但隐藏，HRM 设置项已经合并到 `/manage/system` 下。
