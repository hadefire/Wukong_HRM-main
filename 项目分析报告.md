# Ever人力资源管理系统 (Wukong_HRM) — 项目分析报告

## 一、项目概述

- **项目名称**：Ever人力资源管理系统 (wk_hrm)
- **描述**：前后端分离的 HRM 系统，涵盖组织规划、招聘、员工、考勤、社保、绩效等模块。
- **官网/体验**：http://www.5kcrm.com / https://www.72crm.com
- **说明**：仓库已包含前端打包产物，**只运行后端即可使用**，无需单独打包或运行前端。

---

## 四、配置要点

### 1. 后端配置（hrm-web）

- **主配置**：`hrm/hrm-web/src/main/resources/application.yml`
- **Profile**：默认 `spring.profiles.active: core`，核心逻辑在 `common/common-web/.../application-core.yml`。
- **启动脚本**：72crm.bat 使用 `-Dspring.profiles.include=core,test`，但**项目中未发现 application-test.yml**，若启动报错可改为只使用 `core` 或自建 test 配置。

当前 `application.yml` 中需要按本机环境修改的项：

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `server.port` | 服务端口 | 28080 |
| `spring.datasource.url` | MySQL 地址与库名 | jdbc:mysql://127.0.0.1:3306/wk_hrm_open?... |
| `spring.datasource.username` / `password` | 数据库账号密码 | root / password |
| `spring.redis.host` / `port` / `password` | Redis | 127.0.0.1 / 6379 / 123456 |
| `spring.elasticsearch.uris` / `username` / `password` | ES 地址与认证 | 192.168.1.210:9200 / elastic / xxx |
| `wukong.ids.auth` | Ever ID 中心（登录/鉴权） | appId、clientId、clientSecret、requestUri、redirectUri |
| `wukong.common.upload` | 本地上传目录 | type: local，路径如 D:/wukong |

### 2. Ever ID 与前端静态资源

- 登录依赖**Ever ID 中心**（https://id.72crm.com/），需在后台配置 `wukong.ids.auth` 的 appId、clientId、clientSecret 等。
- `hrm-web/src/main/resources/static/APPLICATION_ID.txt` 内容需与 `wukong.ids.auth.appId` 一致（当前为 1711560062472347648），供前端读取。

### 3. 数据库

- **库名**：`wk_hrm_open`
- **字符集**：utf8mb4
- **初始化**：执行 `DB/wk_hrm_open.sql`（建表、菜单、权限等）。

### 4. 前端开发（可选）

- 前端源码在 `ux/`，开发时：`npm install` → `npm run dev`（默认端口 8090）。
- `ux/vue.config.js` 里 devServer 代理为 `http://localhost:1001/`，而后端默认端口为 **28080**，若直连后端需把代理改为 `http://localhost:28080/` 或与本机实际端口一致。

## 五、依赖与构建注意点

1. **common-web 版本**：根 pom 中 `core.version=2.3.211`，common-web 使用该版本，需从根目录执行 `mvn install`，先安装 common 再构建 hrm。
2. **本地 JAR**：hrm-web 依赖两个 system 作用域 JAR：
   - `src/main/resources/lib/wk_common_web-1.0.6.jar`
   - `src/main/resources/lib/provider-1.0.1.jar`  
   已随仓库提供，无需额外下载。
3. **webapp/public**：pom 中有一处把 `src/main/webapp/public` 复制到 `target/public`，但**项目里没有 webapp 目录**。若执行 `mvn package` 报错“找不到目录”，可：
   - 新建空目录 `hrm/hrm-web/src/main/webapp/public`，或
   - 在 pom 中为该 copy-resources 的 execution 增加 `allowNonExistingSourceDirectory=true`（视 Maven 版本支持情况）。

## 六、部署与运行流程（简要）

### 环境准备

- JDK 1.8、Maven 3.5+、MySQL 8+、Redis、Elasticsearch 8.5.3（按需，若功能未用到可先不配或后续再配）。

### 步骤

1. **数据库**  
   - 创建库：`create database wk_hrm_open character set utf8mb4 collate utf8mb4_general_ci;`  
   - 导入：`use wk_hrm_open; source /path/to/DB/wk_hrm_open.sql;`

2. **修改配置**  
   - 编辑 `hrm/hrm-web/src/main/resources/application.yml`：数据库、Redis、ES（如用）、Ever ID、上传路径等。  
   - 确认 `static/APPLICATION_ID.txt` 与 appId 一致。

3. **构建**  
   - 在**项目根目录**执行：  
     `mvn clean -Dmaven.test.skip=true install`  
   - 再在 `hrm/hrm-web` 打包：  
     `mvn -Dmaven.test.skip=true package`  
   - 若需部署包：使用生成的 `hrm/hrm-web/target/hrm-web.zip`。

4. **运行**  
   - **方式一**：在 `hrm/hrm-web/target` 下执行  
     `java -Dspring.profiles.include=core -jar hrm-web-0.0.1-SNAPSHOT.jar`  
   - **方式二**：解压 `hrm-web.zip` 后，用 72crm.bat（Windows）或 72crm.sh（Linux）启动（脚本内需使用实际 jar 名，打包时 assembly 会替换占位符）。

5. **访问**  
   - 前端页面：http://localhost:28080/（或你配置的 port）  
   - 接口文档：http://localhost:28080/doc.html

## 八、风险与建议

| 项目 | 说明 |
|------|------|
| Ever ID 依赖 | 登录强依赖官方 ID 中心，自建或内网需自备 OAuth2 或改造鉴权。 |
| ES | 部分功能可能依赖 ES，若未安装需在配置或代码中关闭或降级，否则启动/检索可能报错。 |
| application-test | 72crm.bat 使用 `core,test`，无 test 配置时可改为仅 `core` 或新增 application-test.yml。 |
| 前端代理 | 本地跑 ux 时，注意 vue.config.js 的 proxy 端口与后端一致（如 28080）。 |
| 目录缺失 | 若打包报错与 webapp/public 有关，按第五节处理。 |

---

**结论**：本项目为多模块 Maven 工程，后端以 Spring Boot + MyBatis-Plus + Redis（+ ES）为主，前端已打包进 hrm-web 的 static，部署时以**改配置 → 建库导 SQL → 根目录 install → hrm-web package → 运行 jar** 为主线即可。首次运行建议先只配置 MySQL、Redis 和Ever ID，确认能启动再按需加 ES 或调整 profile。
