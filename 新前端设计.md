# 新前端设计

本文不考虑当前技术实现方式，只从不同角色用户的真实使用场景出发，重新设计 HRM 系统的前端功能模块、页面结构与功能点。设计目标是让系统更符合不同角色的工作习惯，减少无关信息干扰，提高日常操作效率。

## 1. 设计目标

### 1.1 总体目标

新的前端不再以“后台模块堆叠”为中心，而是以“角色任务”为中心：

- 系统管理员看到的是配置、权限、安全、流程和系统健康。
- 公司领导看到的是经营视角的人力资源数据、趋势、风险和决策入口。
- 普通员工看到的是自己每天要处理的事项、个人信息、薪资绩效、请假考勤和服务申请。

### 1.2 易用性原则

- 首页只放当前角色最常用、最重要的信息。
- 菜单名称使用业务语言，避免技术或权限名称。
- 同一件事只提供一个明显入口，减少重复菜单。
- 高频操作放在首页或列表右上角。
- 复杂流程拆成步骤向导。
- 对用户而言无权限的模块不展示，而不是展示后提示无权限。
- 所有待办、审批、提醒统一入口。
- 所有列表支持快速搜索、常用筛选、保存筛选条件。
- 关键数据提供解释说明，例如“本月离职率为什么升高”。
- 移动端和 PC 端功能分层：移动端处理轻任务，PC 端处理复杂配置和批量操作。

## 2. 角色划分

### 2.1 系统管理员

典型用户：

- IT 管理员
- 企业系统管理员
- HR 系统维护人员
- 权限与组织配置负责人

核心诉求：

- 快速完成用户、角色、菜单、审批流、基础参数维护。
- 确保系统安全、权限正确、数据可追踪。
- 降低配置复杂度，避免误操作影响业务。

### 2.2 公司领导

典型用户：

- CEO / 总经理
- HRD / 人力资源负责人
- 部门负责人
- 财务负责人

核心诉求：

- 快速了解公司人力现状。
- 看到组织、招聘、离职、薪酬、绩效、考勤等关键趋势。
- 发现风险和异常。
- 快速进入审批和决策动作。

### 2.3 普通员工

典型用户：

- 公司普通员工
- 新员工
- 试用期员工
- 一线主管或小组长

核心诉求：

- 快速打卡、请假、查看工资条、查看社保、查看绩效。
- 及时处理自己的待办和审批。
- 清楚知道个人档案、合同、薪资、绩效的状态。
- 尽量少接触复杂 HR 管理概念。

## 3. 总体信息架构

### 3.1 角色门户

登录后不直接进入统一后台，而是进入角色门户：

- 系统管理员门户
- 公司领导门户
- 员工门户

如果一个用户拥有多个角色，顶部提供“身份切换”：

- 当前身份：员工 / 领导 / 管理员
- 切换后整个菜单和首页随身份变化
- 默认进入最近一次使用的身份
- 支持用户自行设置默认身份

### 3.2 通用顶部区域

所有角色共享顶部能力：

- 全局搜索
- 消息中心
- 待办中心
- 帮助中心
- 个人中心
- 身份切换
- 公司 Logo 与系统名称

### 3.3 通用底层能力

所有角色共享但不一定直接展示的能力：

- 文件上传与预览
- 导入导出
- 审批流
- 操作日志
- 权限控制
- 消息通知
- 数据字典
- 系统帮助

## 4. 系统管理员界面设计

## 4.1 管理员首页

页面名称：

- 系统控制台

首页目标：

- 让管理员一眼知道系统是否正常、是否有待处理配置和安全风险。

首页模块：

- 系统健康概览
  - 今日登录人数
  - 接口异常数量
  - 文件上传失败数量
  - 待处理系统告警
  - Redis / MySQL / 存储连接状态
- 配置完成度
  - 组织架构是否完整
  - 角色权限是否已配置
  - 审批流是否启用
  - 考勤规则是否启用
  - 薪资规则是否启用
  - 社保方案是否启用
- 管理员待办
  - 待开通账号
  - 待重置密码
  - 待处理权限申请
  - 待审核导入错误
  - 待修复异常流程
- 最近变更
  - 最近新增管理员
  - 最近角色权限调整
  - 最近菜单变更
  - 最近审批流发布
  - 最近系统参数变更
- 快捷入口
  - 新增员工账号
  - 配置角色权限
  - 配置审批流
  - 导入组织架构
  - 查看操作日志
  - 系统备份

易用性设计：

- 首页卡片提供“立即处理”按钮。
- 异常项用红色、风险项用黄色、正常项用绿色。
- 每个配置项显示“完成 / 未完成 / 有风险”。
- 对新部署系统提供“初始化向导”。

  - 职级体系
  - 岗位序列

- 组织变更模拟器：变更前预览影响人数、角色、审批链。
- 离职账号自动回收：员工离职后自动禁用账号并回收权限。
- 异常账号提醒：长期未登录、多地登录、管理员权限过高提醒。

  - 招聘延期职位
  - 考勤异常高发部门

## 11. 设计后的用户路径示例

### 11.1 系统管理员配置新公司

1. 登录进入系统控制台。
2. 首页提示“初始化未完成”。
3. 点击初始化向导。
4. 导入组织架构。
5. 创建管理员账号。
6. 配置角色权限。
7. 配置审批流。
8. 配置考勤、薪资、社保、绩效基础规则。
9. 系统提示配置完成度 100%。
10. 发布给员工使用。

### 11.2 公司领导查看本月人力情况

1. 登录进入人力驾驶舱。
2. 查看本月在职人数、入离职、薪资成本、绩效达标率。
3. 发现某部门离职率异常。
4. 点击风险提醒进入组织洞察。
5. 查看该部门人员结构、离职趋势和绩效情况。
6. 指派 HR 负责人跟进。
7. 进入决策审批中心处理待审批事项。

### 11.3 普通员工处理一天工作事项

1. 登录进入我的工作台。
2. 查看今日班次和打卡状态。
3. 首页提示有一个绩效目标待确认。
4. 点击待办进入我的绩效。
5. 查看目标并确认。
6. 午后需要请假，点击快捷操作“请假”。
7. 填写请假申请并提交。
8. 下班后收到工资条提醒。
9. 二次验证后查看工资条并确认。

## 12. 与现有前端模块的关系

现有系统已有较完整的 HRM 功能基础，新设计主要是重新组织用户界面：

- 将原“HRM 工作台”升级为按角色区分的首页。
- 将原“系统管理”拆成更清晰的管理员任务模块。
- 将原“员工端”扩展为完整员工门户。
- 将领导角色从原管理端中独立出来，提供数据驾驶舱与决策中心。
- 将审批、消息、待办统一收口，减少用户在多个模块间寻找事项。
- 将原有薪资、社保、绩效等功能按用户角色拆分展示深度。

## 13. 优先级建议

### 第一阶段：信息架构重构

- 增加角色门户。
- 调整三类角色菜单。
- 重做三个角色首页。
- 统一待办中心。
- 统一消息中心。

### 第二阶段：体验优化

- 列表筛选优化。
- 表单向导化。
- 工资条、绩效、考勤说明增强。
- 审批摘要和风险提示。
- 敏感数据脱敏和二次验证。

### 第三阶段：智能分析

- 人力驾驶舱。
- 组织健康分。
- 招聘风险预测。
- 薪资成本分析。
- 绩效人才盘点。
- 管理员权限体检。

## 14. 总结

新的前端应该从“功能堆叠型后台”升级为“角色任务型门户”。

系统管理员关注稳定、安全、配置和权限；公司领导关注数据、趋势、风险和审批；普通员工关注自己每天要完成的任务、个人权益和申请进度。通过角色化首页、清晰菜单、统一待办、统一消息、敏感数据分级和更多解释型设计，可以显著降低使用门槛，也能让 HRM 系统更贴近日常工作场景。
